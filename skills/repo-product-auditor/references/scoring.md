# Repo Product Auditor Scoring

Score each category from 1 to 5.

## 1. Positioning

1 = unclear what the repo is  
3 = understandable but generic  
5 = specific, memorable, audience-aware  

## 2. README clarity

1 = hard to scan  
3 = basic structure exists  
5 = clear hero, quick start, examples, use cases  

## 3. Visual proof

1 = no screenshot/demo  
3 = some output shown  
5 = strong screenshot/GIF/demo/diagram  

## 4. Setup friction

1 = hard to install/run  
3 = basic commands exist  
5 = copy-paste quick start works  

## 5. Product credibility

1 = looks unfinished  
3 = usable but thin  
5 = feels maintained, intentional, tested, useful  

## 6. Discovery readiness

1 = poor title/topics/description  
3 = basic GitHub metadata  
5 = strong topics, description, README keywords, demo, release  

## 7. Portfolio value

1 = does not support the author's positioning  
3 = somewhat relevant  
5 = clearly strengthens professional profile  

## 8. Next-action clarity

1 = visitor does not know what to do  
3 = some links or commands exist  
5 = obvious next action: run, demo, star, read case, install  

## Score interpretation

32-40 = strong product-level repo  
24-31 = good repo, needs polish  
16-23 = promising but unclear  
8-15 = needs repositioning before promotion
