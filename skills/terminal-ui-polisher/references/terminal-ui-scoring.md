# Terminal UI Scoring

Score each area from 1 to 5.

## 1. Visual hierarchy

1 = hard to scan  
3 = sections exist but inconsistent  
5 = clear hierarchy and calm layout  

## 2. Menu clarity

1 = unclear options  
3 = usable but crowded  
5 = grouped, obvious, consistent  

## 3. Keyboard flow

1 = confusing or missing navigation  
3 = basic navigation  
5 = predictable shortcuts and return behavior  

## 4. Status feedback

1 = silent or vague  
3 = basic success/error  
5 = clear state model: ready/running/success/warning/failed  

## 5. Error handling

1 = raw errors only  
3 = some friendly messages  
5 = clear cause and recovery command  

## 6. Consistency

1 = every screen behaves differently  
3 = mostly consistent  
5 = same layout grammar everywhere  

## 7. Terminal resilience

1 = breaks with width/path/color issues  
3 = works in normal cases  
5 = handles narrow widths, no-color, long paths, missing tools  

## 8. Product feel

1 = script dump  
3 = useful but plain  
5 = feels intentional and memorable  

## Score interpretation

32-40 = product-level terminal UI  
24-31 = good, needs polish  
16-23 = usable but inconsistent  
8-15 = needs structural redesign
